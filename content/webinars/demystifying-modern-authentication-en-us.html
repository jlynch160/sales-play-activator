Demystifying Modern Authentication

MajorKey Technologies Webinar Modern Authentication Zero Trust Protocol Migration

Summary

An educational session breaking down how modern authentication and authorization works with Microsoft Entra ID. Covers the fundamentals of token-based authentication, the differences between SAML, OAuth 2.0, and OpenID Connect, and how conditional access policies enforce zero trust. Practical guidance on migrating from legacy authentication (NTLM, Kerberos) to modern protocols.

Key Takeaways

  • Modern authentication is fundamentally about getting a token - the application requests it, Entra ID authenticates the user and issues a token with claims
  • SAML handles enterprise SSO, OAuth 2.0 handles delegated authorization, and OpenID Connect adds identity on top of OAuth - each serves a different purpose
  • Conditional access policies are the enforcement engine of zero trust - they evaluate signals (device, location, risk) before granting access
  • Migrating from NTLM and Kerberos to modern authentication requires application discovery and protocol mapping before cutover
  • B2C identity integration is a growing use case alongside workforce authentication - the same Entra platform handles both
