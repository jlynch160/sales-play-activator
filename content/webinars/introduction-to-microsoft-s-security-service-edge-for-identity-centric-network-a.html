Introduction to Microsoft SSE for Identity-Centric Network Access

MajorKey Technologies Webinar Security Service Edge Identity-Centric Access Zero Trust Network

Summary

An introductory session on Microsoft's Security Service Edge and how it enables identity-centric network access. Covers the shift from perimeter-based security to identity-driven access, Global Secure Access architecture, and how SSE integrates with the broader Microsoft security stack.

Key Takeaways

  • SSE shifts network security from perimeter-based (firewalls, VPNs) to identity-based - access decisions are made per user, per application
  • Global Secure Access acts as the traffic management plane connecting users to both private applications and internet resources
  • The identity-centric model means the same conditional access policies protecting SaaS apps now protect on-premises and private network resources
  • SSE eliminates the need for traditional VPN infrastructure which is expensive to maintain and provides overly broad network access
  • Integration with Microsoft Defender provides real-time threat detection on network traffic with automatic session termination for compromised users
